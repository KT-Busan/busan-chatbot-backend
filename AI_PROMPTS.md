# 핵심 AI 프롬프트 (초안)

> ⚠️ **주의**: 이 문서는 실제로 사용했던 프롬프트 원본이 아니라, `busan-chatbot-backend`/`busan-chatbot-frontend` 두 저장소의 **커밋 히스토리와 코드 변경 패턴을 근거로 재구성한 대표 프롬프트 초안**입니다. K-AI Contents Award 지원서에는 "실제 사용한 핵심 AI 프롬프트"를 기재해야 하므로, 제출 전 반드시 본인이 실제로 사용했던 프롬프트로 검토·수정하시기 바랍니다. (당시 대화 세션 로그가 남아있다면 그것을 우선 사용하세요.)

아래는 커밋 로그(`feat(backend): 회의 요청 사항 기반 버튼 로직 전면 개편`, `refactor/BE-refactoring-and-security-hardening`, `fix(backend): 버튼 답변 생성 로직 오류 수정`, `refactor: API 중앙화, 환경변수화` 등)에서 유추한, 개발 단계별로 사용했을 법한 대표 프롬프트입니다.

## 1. 설계 단계

> "부산시 16개 구·군의 청년공간/프로그램 정보를 안내하는 챗봇을 만들려고 해. Flask 백엔드 + React 프론트엔드 구조로, 크롤링한 데이터와 운영자가 수동으로 정리한 override 데이터를 병합해서 보여주고 싶어. 전체 아키텍처와 폴더 구조를 제안해줘."

## 2. 데이터 수집(크롤러) 구현

> "BeautifulSoup으로 [부산청년플랫폼] 사이트에서 청년공간 목록(이름, 지역, 주소, 연락처, 운영시간, 링크)을 크롤링하는 파이썬 코드를 작성해줘. 사이트 구조가 바뀌어도 크롤러 전체가 죽지 않고, 실패 시 마지막으로 성공한 캐시 데이터를 반환하도록 예외 처리를 포함해줘."

## 3. Override 병합 로직

> "크롤링 데이터와 운영자가 직접 수정한 override JSON 데이터를 이름 기준으로 병합하는 함수를 만들어줘. override에 있는 항목은 크롤링 데이터를 덮어쓰고, override에만 있는 신규 항목은 추가해야 해."

## 4. 챗봇 응답 로직 리팩토링

> "지금 챗봇 응답 로직이 버튼 텍스트별 if-elif가 너무 길어서 유지보수가 어려워. 버튼/키워드/지역/자유 대화(OpenAI GPT-4o) 순서로 우선순위를 갖는 라우팅 구조로 리팩토링해줘. 응답 형식(JSON)도 `{success, data, message}`로 통일해줘."

## 5. API 응답 형식 표준화 및 Blueprint 분리

> "Flask 라우트가 app.py 한 파일에 다 몰려 있어. 기능별(chat, user, space, program)로 Blueprint를 분리하고, 공통 에러 응답 핸들러를 만들어서 404/400/500을 일관된 JSON 형식으로 반환하도록 리팩토링해줘."

## 6. 보안/환경변수 점검

> "이 Flask 프로젝트에서 API 키나 DB 접속 정보가 코드에 하드코딩되어 있는지 점검하고, 전부 환경변수(.env)로 옮겨줘. CORS도 와일드카드가 아니라 프론트엔드 도메인만 허용하도록 화이트리스트로 바꿔줘."

## 7. 프론트엔드 API 연동 중앙화

> "axios 호출 코드가 여러 컴포넌트에 흩어져 있어서 백엔드 URL이나 타임아웃을 바꿀 때마다 여러 파일을 고쳐야 해. `src/api/index.js` 하나로 API 호출을 중앙화하고, 백엔드 URL은 환경변수(`VITE_BACKEND_URL`)로 관리하도록 리팩토링해줘."

## 8. 버그 수정 (디버깅)

> "지역 검색 버튼을 눌렀는데 '청년공간을 찾을 수 없습니다'라는 응답이 나와. 프론트엔드에서 보내는 지역명과 백엔드가 기대하는 지역명 형식이 다른 것 같은데, 요청/응답을 같이 보고 원인을 찾아줘."

## 9. UI/다크모드 일관성

> "CSS 변수(`variables.css`)로 라이트/다크 모드 색상 토큰을 정의했는데, 일부 컴포넌트는 이 변수를 안 쓰고 색상을 직접 하드코딩하고 있어. 다크모드에서 대비가 깨지는 부분이 있는지 점검하고, 변수를 쓰도록 통일해줘."

## 10. 배포 안정화

> "Render 무료 티어라 백엔드가 일정 시간 요청이 없으면 슬립 상태가 되고, 첫 요청이 느려. 프론트엔드에서 로딩 인디케이터와 타임아웃/에러 메시지를 자연스럽게 보여주도록 개선해줘."
