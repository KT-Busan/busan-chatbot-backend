# 문제 정의 및 해결 방안 (지원서 초안)

> ⚠️ 아래 초안은 코드/README/커밋 히스토리를 근거로 작성되었습니다. 구체적인 수치(설문 결과, 실제 이용 통계 등)나 개인적인 개발 계기가 있다면 반영하여 수정한 뒤 지원서에 사용하시기 바랍니다.

## 어떤 생활 속 불편함을 해결했는가

부산에는 청년을 위한 공간과 프로그램이 16개 구·군, 8개 카테고리(스터디/회의, 교육/강연, 커뮤니티, 진로/창업, 문화/창작, 작업/창작실, 휴식/놀이, 행사/이벤트)에 걸쳐 이미 다양하게 마련되어 있습니다. 그러나 이 정보는 부산광역시 공식 채널, 각 구·군 청년센터, 개별 기관 홈페이지·SNS 등에 흩어져 있어, 정작 이용하려는 청년은 "내가 사는 지역에, 내가 원하는 목적으로 쓸 수 있는 공간이 어디 있는지"를 알아보기 위해 여러 사이트를 일일이 찾아다녀야 합니다. 특히 처음 청년공간을 이용해보려는 사람일수록 어떤 카테고리의 공간이 있는지조차 파악하기 어렵고, 이 정보 격차는 결국 이미 마련된 공공 자원이 충분히 활용되지 못하는 결과로 이어집니다. B-BOT은 이 "정보는 있지만 찾기 어렵다"는 문제를 해결하기 위해 만들어졌습니다.

## AI를 어떻게 활용했는가

B-BOT은 OpenAI GPT-4o를 대화 엔진으로 사용하여, 사용자가 자연어로 질문하면("서면 근처에서 모임할 수 있는 곳 있어?") 정해진 버튼 흐름 없이도 원하는 정보에 접근할 수 있도록 설계했습니다. 지역명·카테고리명 등 구조화된 질의는 빠른 키워드/지역 매칭으로 즉시 응답하고, 그 외의 자유로운 질문은 GPT-4o가 이전 대화 맥락을 참고해 자연스럽게 응답을 생성하는 이중 구조를 사용합니다. 또한 개발 과정 전반(백엔드 리팩토링, 크롤러 예외 처리 설계, API 응답 형식 표준화, 보안 점검, 프론트엔드 컴포넌트 구조화 등)에서 Claude Code를 활용해 코드 설계·구현·디버깅·리팩토링을 반복적으로 수행했습니다. 이를 통해 1인 개발 체제에서도 백엔드(Flask)와 프론트엔드(React) 양쪽을 일관된 구조로 유지하며 빠르게 기능을 반복 개선할 수 있었습니다.

## 서비스가 실제로 하는 일

1. **정보 수집**: BeautifulSoup 크롤러가 부산청년플랫폼의 청년공간·프로그램 정보를 주기적으로 수집합니다.
2. **정확도 보정**: 크롤링만으로 놓치기 쉬운 최신 정보는 운영자가 직접 정리한 override 데이터와 자동 병합되어, 정확한 정보를 우선 노출합니다.
3. **대화형 탐색**: 사용자는 메뉴 버튼 또는 자유 문장으로 지역·카테고리·조건별 공간을 검색하고, 지도에서 지역별 공간 개수를 한눈에 확인하며, 상세 정보와 대관 신청 링크까지 이어지는 흐름으로 정보를 얻습니다.
4. **안정적인 서비스 운영**: 크롤링이 실패해도 마지막으로 성공한 캐시 데이터로 자동 폴백하여 서비스 중단 없이 정보를 계속 제공합니다.

## 향후 발전 가능성

카카오톡 채널 연동으로 접근성을 더 낮추고, 이용 이력 기반 개인화 추천으로 탐색 시간을 더 줄이며, 부산시 공공데이터 API 정식 연동으로 데이터 신뢰성을 높이고, 장기적으로는 청년공간뿐 아니라 지역 소상공인·일자리 정보까지 다루는 "부산 청년 생활 정보 허브"로 확장할 수 있습니다.
